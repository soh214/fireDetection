# 3월 20일 회의 회의록

- 녹음 파일: `26년3월20일.m4a`
- 회의 길이: 약 13분 43초
- 주제: VLM/CLIP 기반 화재 탐지 시스템의 구체적 방향 및 산출물 범위 정리

## 1. 회의 개요

이번 회의에서는 화재 탐지 프로젝트의 기술 구조와 학생 팀이 한 학기 동안 수행할 수 있는 범위를 다시 정리했다. 프로젝트는 단순히 화재 후보를 탐지하는 데서 끝나는 것이 아니라, 탐지된 영역이 실제 화재인지 아닌지를 문맥 기반으로 재판별하는 2-stage 구조로 설명되었다.

1차 단계에서는 YOLO와 같은 일반적인 객체 탐지 모델로 화재 후보 영역을 찾고, 2차 단계에서는 해당 bounding box 영역을 CLIP/VLM에 입력하여 화재 관련 attention 또는 text key 값을 기준으로 실제 화재 여부를 판단하는 방향이 제안되었다.

## 2. 주요 논의 내용

### 기술 방향

- 산업 현장에서는 단순 화재 탐지보다 “탐지 후 실제 화재인지 재확인하는 단계”에 대한 수요가 많다.
- 전체 구조는 2-stage 방식으로 정리되었다.
  - 1단계: YOLO 등 detection 모델로 화재 후보 영역 탐지
  - 2단계: 탐지된 bounding box 영역을 CLIP/VLM에 입력해 실제 화재 여부 재판별
- YOLO와 CLIP 자체는 기존 모델을 활용하면 되므로, 핵심 작업은 화재 판단에 사용할 text key 값 또는 attention 기준을 어떻게 정의할지에 있다.
- 여러 화재 이미지 데이터를 모델에 넣었을 때 공통적으로 나타나는 텍스트/attention 값이 무엇인지 먼저 정의해야 한다.
- 해당 기준이 잡히면 이후 개발 및 시스템 연결은 상대적으로 큰 공수가 들지 않을 것으로 보았다.

### 데이터셋

- 화재 이미지 데이터는 AI Hub와 해외 화재/연기 데이터셋을 활용할 수 있다.
- 필요한 경우 멘토를 통해 참고 가능한 데이터셋 정보를 공유받기로 했다.
- 학생 팀은 화재 이미지뿐 아니라 실제 화재가 아닌 상황과 비교할 수 있는 데이터도 함께 고려해야 한다.

### 수행 범위

- 한 학기 단기 프로젝트이므로 전체 온디바이스 구현이나 모델 경량화까지 수행하기는 어렵다.
- 현재 단계에서는 PC 또는 workstation 기준으로 동작하는 시스템을 목표로 보면 된다.
- 온디바이스 적용을 하려면 모델 경량화가 필요하지만, 이는 이번 학기 범위보다는 후속 연구 주제로 보는 것이 적절하다.
- 이번 학기에는 화재 탐지를 위한 VLM/CLIP 활용 구조를 구현하고, attention/text key 기준을 정의하는 데 집중한다.

### 논문 가능성

- VLM을 활용해 화재 탐지를 수행하는 구조 자체가 논문 주제가 될 수 있다고 설명되었다.
- 특히 YOLO 단독 탐지와 달리 CLIP/VLM의 화재 관련 attention 값을 정의하고 threshold를 설정하는 방식은 연구자마다 접근이 달라질 수 있어 비교 연구가 가능하다.
- 기존 모델 기반 결과와 VLM 기반 재판별 결과를 비교하면 논문으로 정리할 수 있는 여지가 있다.

### 소프트웨어 등록

- 소프트웨어 등록은 모델 하나만 등록하기보다는 사용자가 볼 수 있는 하나의 시스템 단위로 등록하는 것이 일반적이다.
- 입력 이미지/영상, YOLO 탐지 결과, CLIP/VLM 재판별 결과, UI 또는 결과 표시 화면까지 묶어 하나의 시스템으로 등록하는 방향이 적절하다고 논의되었다.
- 전체 시스템을 하나로 등록해도 되고, 끊을 수 있는 부분이 명확하다면 별도 모듈로 나누어 등록할 수도 있다.
- 최종적으로는 개발 환경, 실행 화면, 결과 표시가 포함된 형태를 고민해야 한다.

### GitHub 공개 여부

- 수업 과정상 GitHub에 진행 과정을 올려야 하지만, public으로 공개할지 private으로 둘지는 별도 확인이 필요하다.
- 이 부분은 멘토 및 관련 담당자와 상의해서 정리하기로 했다.

### VLM 경험의 의미

- 화재 탐지는 이번 프로젝트의 적용 분야일 뿐, VLM을 활용해 인지 단계를 구성하는 경험 자체가 중요하다고 설명되었다.
- 향후 온디바이스, 모바일, 로봇, 피지컬 AI 등 다양한 분야에서도 VLM 기반 인지 단계가 중요하게 사용될 수 있다.
- VLM을 다뤄보는 경험은 향후 취업이나 다른 연구 주제로 확장하는 데 도움이 될 것으로 언급되었다.
- 후속 연구로는 화재 탐지용 경량 VLM 또는 경량화된 VLM 기반 시스템을 고려할 수 있다.

## 3. 결정 사항

- 프로젝트는 YOLO 기반 1차 탐지와 CLIP/VLM 기반 2차 재판별 구조로 진행한다.
- 학생 팀은 온디바이스 구현보다 PC/workstation 환경에서 동작하는 시스템 구현을 우선한다.
- 핵심 과제는 화재 이미지에서 공통적으로 나타나는 text key/attention 값을 정의하고, 이를 실제 화재 판단 기준으로 활용하는 것이다.
- 데이터셋은 AI Hub 및 해외 화재/연기 데이터셋을 조사하고, 필요 시 멘토를 통해 추가 정보를 받는다.
- 소프트웨어 등록은 모델 단독이 아니라 UI와 결과 표시를 포함한 시스템 단위로 준비하는 방향이 적절하다.
- GitHub 공개 범위는 추후 멘토 및 담당자와 확인한다.
