# 4월 29일 회의 회의록

- 녹음 파일: `26년4월29일.m4a`
- 회의 길이: 약 17분 37초
- 주제: YOLO+CLIP 기반 화재 재판별 실험 결과 검토, 논문 작성 방향, 소프트웨어 등록 준비

## 1. 회의 개요

이번 회의에서는 기존에 설계한 시스템 아키텍처를 바탕으로 구현한 YOLO+CLIP 결합 구조의 실험 결과를 공유하고, 이를 논문에서 어떤 방식으로 설명할지 논의했다. 학생 팀은 먼저 YOLO로 화재/연기 후보 영역을 탐지하고, 해당 이미지 또는 bounding box 결과를 CLIP에 넣어 VQA/VLM 기반으로 만든 caption들과 비교하는 코드를 작성했다.

실험 예시로 요리 이미지와 벽난로 이미지 데이터를 사용했으며, YOLO가 불로 탐지한 결과 중 CLIP 단계에서 실제 위험한 화재 여부를 다시 줄여내는 방향의 가능성을 확인했다.

## 2. 주요 논의 내용

### 구현 현황

- 시스템은 기존 아키텍처대로 `YOLO 1차 탐지 → CLIP 기반 caption 유사도 비교 → 실제 위험 화재 여부 판단` 흐름으로 구현되었다.
- YOLO는 이미지에서 fire/smoke bounding box와 confidence를 출력하는 방식으로 사용되었다.
- 이후 YOLO가 탐지한 이미지 또는 영역과, VQA/VLM 모델로 생성한 caption을 CLIP에 함께 넣어 가장 유사한 caption을 선택하도록 했다.
- 이를 통해 단순히 불이 보이는지 여부가 아니라, 위험한 화재인지 아닌지를 재판별하는 구조를 만들었다.

### 실험 결과

- 요리 이미지 약 200장을 대상으로 테스트했을 때, YOLO가 불을 탐지한 이미지에 대해 CLIP 단계에서는 위험한 화재가 없다는 방향으로 판단되었다.
- 벽난로 이미지의 경우 YOLO에서 174개 정도가 fire로 탐지되었고, CLIP 재판별 후 위험한 불로 분류된 것은 2개 정도로 줄었다.
- 이 결과는 YOLO 단독 사용 시 불처럼 보이는 장면을 과도하게 화재로 잡을 수 있고, CLIP을 결합하면 false positive를 줄일 수 있다는 주장에 활용할 수 있다.

### YOLO를 사용하는 이유

- “caption이 이미 있다면 YOLO 없이 CLIP만 사용해도 되는 것 아닌가”라는 질문이 제기되었다.
- 이에 대해 YOLO는 object detection 모델이므로, 이미지 내에서 화재 후보 영역을 찾기 위해 필요하다고 설명되었다.
- CLIP은 detection 모델이 아니며, 이미지와 텍스트의 유사도를 비교하는 역할에 가깝다.
- 따라서 YOLO는 후보 영역 탐지 및 bounding box guidance를 제공하고, CLIP은 해당 후보가 실제 위험 상황인지 문맥적으로 재판별하는 역할로 구분된다.

### Ablation study 논의

- 더 학술적으로 설득력 있는 논문을 만들려면 `YOLO 제거`, `CLIP 제거`, `YOLO+CLIP 결합`을 비교하는 ablation study를 수행하는 것이 좋다는 의견이 있었다.
- 예를 들어 YOLO를 빼고 전체 이미지와 caption만 CLIP에 넣었을 때 오탐이 얼마나 늘어나는지 비교하면, YOLO를 함께 쓰는 이유를 더 명확히 설명할 수 있다.
- 다만 학부생 논문 경진대회 수준에서는 YOLO 단독과 YOLO+CLIP 결합 비교만으로도 큰 무리는 없다고 언급되었다.
- 가능하다면 YOLO 없이 CLIP만 사용한 결과도 추가 실험하여 논문 완성도를 높이는 방향이 좋다.

### 논문 주장 방향

- 기본 주장은 “YOLO 단독 화재 탐지는 불처럼 보이는 비화재 상황을 많이 탐지하지만, CLIP 기반 문맥 재판별을 결합하면 오탐을 줄일 수 있다”로 정리되었다.
- 예시로 YOLO 단독에서는 200장 중 172개를 fire로 탐지했지만, YOLO+CLIP 결합 후 위험 화재로 판단된 수가 크게 줄었다는 결과를 제시할 수 있다.
- 논문에서 더 설득력 있게 쓰려면 단순 개수 비교뿐 아니라 오탐률, 탐지율, AUC 등 평가 지표를 찾아보고 적용하는 것이 좋다는 피드백이 있었다.

### 논문 일정

- 학교 논문 제출 마감은 5월 8일 금요일로 언급되었다.
- 논문은 최소 2페이지, 최대 5페이지 정도의 형식으로 보인다.
- 초안은 수요일까지 멘토에게 전달하고, 멘토가 하루 정도 검토한 뒤 목요일에 수정본을 반영하여 금요일 제출하는 일정이 제안되었다.
- 가능한 한 빨리 초안을 작성해 피드백을 받는 것이 좋다.

### 소프트웨어 등록

- 소프트웨어 등록은 논문 작성 이후 진행하기로 했다.
- 등록 절차는 대학원/연구실을 통해 진행하며, 보통 2주에서 한 달 정도 걸릴 수 있다고 언급되었다.
- 멘토가 필요한 입력 항목을 정리해서 전달하면, 학생 팀이 해당 내용을 채워 제출하는 방식으로 진행할 수 있다.
- 웹 UI나 별도 사용자 인터페이스가 반드시 필요한 것은 아니며, 코드 압축 파일과 간단한 아키텍처/설명 자료만으로도 등록이 가능할 수 있다고 설명되었다.

## 3. 결정 사항

- 논문 기본 방향은 YOLO 단독 대비 YOLO+CLIP 결합 방식이 오탐을 줄인다는 주장으로 잡는다.
- YOLO는 화재 후보 영역 탐지, CLIP은 문맥 기반 재판별 역할로 구분해 설명한다.
- 가능하면 YOLO 없이 CLIP만 사용하는 비교 실험도 추가해 ablation study 형태로 보강한다.
- 평가 지표로 오탐률, 탐지율, AUC 등 적절한 metric을 조사해 논문에 반영한다.
- 논문 초안은 수요일까지 멘토에게 전달하고, 금요일 마감 전에 수정해 제출한다.
- 소프트웨어 등록은 논문 이후 진행하며, UI 없이 코드와 설명 자료 중심으로 준비한다.

- 요리/벽난로 등 비화재 불 장면에서 오탐 감소 효과 정리
- 오탐률, 탐지율, AUC 등 사용할 평가 지표 조사
- 논문 초안 작성 및 수요일까지 멘토에게 전달
- 논문에 시스템 아키텍처, 실험 데이터, 실험 결과, 한계점 포함
- 소프트웨어 등록을 위해 코드와 아키텍처 설명 자료 정리
